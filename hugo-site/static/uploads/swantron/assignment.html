Part III

Assume that you want automate clicking a button in a browser. There is no way to grab a reference to the button; the only way to determine where the button is, is by an event that is triggered by the cursor hovering over it. Write a function that can return any coordinate within the button by moving the mouse around the screen until the button's hover event is triggered'


I spun off on a tangent working on a solution to this. Here is the basic idea of my plan-of-attack and where I went with this problem.


  • Initial thought: create a “walker” function that moves around the browser, listens for a hover event, and spits results somewhere.

  • Primary issue to overcome: Traditional WebDriver handling of this would require knowing the button’s ID

  • Ideal solution: use Selinium IDE / Jquery to inject and control web page (incomplete)

  • Dirty solution: script around the issue by adding ID

  • Coordinate Grabber: injected script to bypass need for ID

    2. Listen for mouse hover event

    3. Assign event Boolean for each point

    4. Log #3

  3. Interpret boundaries

  4. Click on button by clicking center of determined bounds

  5. Ensure button click event

Summary


This is where I ran out of time. The component I’m missing is the final step...to create the stepper function to walk the page and spit out the bounds. The coordinate grabber unit test shows that I have the events defined to do so, and the exercise becomes determining how to best do this. My next step was to hook on to the body element, and iterate through the page pixel-by-pixel, until the onmouseover event is fired. I would then walk the X-axis until the onmouseout event fires, back up a pixel, and walk the Y-axis until the onmouseout event is fired. This would provide the bounds used to do the clicking.
